👨‍💻 Code-Together 14: Non-Comparison Sorts

🎯 Overview

An in-class code-together activity introducing non-comparison sorting algorithms — counting sort, bucket sort, and radix sort. Unlike CT12 and CT13’s comparison-based sorts, these algorithms never compare two elements directly. Instead, they exploit integer key properties (value range, digit structure) to achieve O(n + k) time — faster than the O(n log n) lower bound for comparison sorts. Students implement all three sorts in NonComparisonSorts.cpp using the patterns walked through in class.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Implement counting sort: find min/max, allocate a count array of size k, tally occurrences, place elements directly
  • Implement bucket sort: distribute elements into range-based buckets, sort each bucket, concatenate
  • Implement radix sort (LSD): sort digit-by-digit from least significant to most, using stable counting sort at each pass
  • Explain what k means for each algorithm and why it determines whether non-comparison sorts are practical
  • Explain why stability is critical in radix sort — unstable sub-sorts would scramble earlier digit orderings
  • Know when to use non-comparison sorts vs comparison sorts (integers with known range vs general data)

📚 Teaching Order

Work through the files in order:

1. NonComparisonSorts.h — Function declarations

Walk through the header. Compare to CT12/CT13: same interface (vector<int>&), but these sorts never use < or > to compare elements.

Section What You’ll Learn
counting_sort() O(n+k) time, O(k) space, stable. Count occurrences, place directly. k = value range.
bucket_sort() O(n+k) avg, O(n²) worst, O(n+k) space. Distribute into buckets by range, sort each, concatenate.
radix_sort() O(d(n+k)) time, O(n+k) space, stable. Sort digit-by-digit using stable counting sort.

2. NonComparisonSorts.cpp — Sort implementations (3 TODOs)

Section What You’ll Learn
TODO 1: counting_sort() Step 1a: find min/max. Step 1b: allocate count array of size k. Step 1c: tally with count[val - min]++. Step 2: walk counts left-to-right, place values.
TODO 2: bucket_sort() Step 1a: find range, compute range_per_bucket. Edge guards (size ≤ 1, min == max). Step 1b: distribute with (val - min) / range_per_bucket. Step 2: sort each bucket. Step 3: concatenate.
TODO 3: radix_sort() Step 0: find max to determine d (number of digits). Main loop: for each digit position, call counting_sort_by_digit. Helper uses backwards iteration for stability.

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct14-non-comparison-sorts.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ? SEE DIAGRAM: comments in each file — these point you to visual walkthroughs in the images/ folder that show exactly what each step does.
